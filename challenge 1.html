Análisis de Tráfico de Red

Presentación de Solución Técnica y Entregables

Visión General del Proyecto

Este reporte detalla la solución desarrollada para el desafío de análisis de tráfico de red. El producto final es una robusta aplicación de línea de comandos construida con Python y Scapy, diseñada para capturar, almacenar y analizar paquetes de red de manera eficiente.

La arquitectura se centra en la portabilidad y la facilidad de uso, empaquetando toda la aplicación y sus dependencias en un contenedor Docker. Esto asegura un despliegue consistente y sin fricciones en cualquier entorno.

Componentes Clave de la Solución

🚀 Captura de Paquetes

Utiliza Scapy para escuchar en una interfaz de red y capturar paquetes en tiempo real, extrayendo metadatos esenciales.

💾 Almacenamiento Persistente

Los datos de los paquetes se almacenan en una base de datos SQLite, una solución ligera y eficiente perfecta para esta escala.

📈 Análisis y Estadísticas

Procesa los datos almacenados para generar un resumen estadístico claro, incluyendo el tráfico total, desglose por protocolo y las IPs más activas.

Documentación Técnica

Stack Tecnológico y Justificación

🐍 Python & Scapy

Elección principal por su madurez en el ámbito de redes y la simplicidad que ofrece Scapy para la manipulación de paquetes.

🗃️ SQLite

Ideal por su naturaleza serverless. Elimina la necesidad de un servicio de base de datos separado, simplificando la contenerización.

🐳 Docker

Garantiza un entorno 100% reproducible y aísla la aplicación, facilitando el despliegue y la ejecución en cualquier sistema.

Esquema de la Base de Datos

Se utiliza una única tabla packets para registrar toda la información necesaria de forma estructurada y optimizada para las consultas requeridas.

Gestión de Datos con SQLite

Para el almacenamiento persistente de los paquetes capturados, se seleccionó SQLite como motor de base de datos. Esta elección estratégica se basa en la simplicidad, eficiencia y perfecta integración con la arquitectura del proyecto.

¿Por qué SQLite?

🚀 Serverless y Autocontenida

SQLite no requiere un proceso de servidor separado. La base de datos completa reside en un único archivo (traffic.db), lo que simplifica enormemente el despliegue y la gestión dentro del contenedor Docker.

🧩 Configuración Cero

No hay necesidad de configurar usuarios, permisos o puertos. La base de datos funciona "out-of-the-box", permitiendo que la aplicación se centre en su lógica principal.

⚡ Ligera y Eficiente

Para el volumen de datos y el tipo de consultas requeridas por el desafío, SQLite ofrece un rendimiento excelente sin consumir recursos innecesarios del sistema.

Esquema Visual y Funcional

La estructura de la base de datos se compone de una única tabla llamada packets, diseñada para almacenar la información esencial de cada paquete de forma eficiente.

Tabla: packets

  • id INTEGER PK
    Identificador único autoincremental.
  • timestamp TEXT
    Fecha y hora de la captura.
  • source_ip TEXT
    Dirección IP de origen.
  • destination_ip TEXT
    Dirección IP de destino.
  • protocol TEXT
    Protocolo (TCP, UDP, ICMP).
  • size INTEGER
    Tamaño del paquete en bytes.

Ciclo de Vida de los Datos

1

Inicialización

Al ejecutar el programa, se conecta al archivo data/traffic.db. Si el archivo o la tabla packets no existen, se crean automáticamente.

2

Inserción en Tiempo Real

Por cada paquete capturado por Scapy, la función packet_handler extrae los datos y los inserta como una nueva fila en la tabla packets.

3

Consulta y Agregación

Una vez finalizada la captura, se ejecutan consultas SQL sobre la tabla packets para calcular las estadísticas. Estas consultas utilizan funciones de agregación como COUNT(*) y cláusulas GROUP BY para generar el reporte final.

Guía de Despliegue y Ejecución

La aplicación está diseñada para ser ejecutada dentro de un contenedor Docker. Siga estos pasos para ponerla en marcha.

Paso 1: Preparación del Entorno

1

Asegúrese de tener Docker instalado y funcionando en su sistema. Luego, cree un directorio para el proyecto y descargue los 4 archivos necesarios (`database.py`, `sniffer.py`, `requirements.txt` y el `Dockerfile` que se muestra a continuación).

Paso 2: Construcción de la Imagen Docker

2

Abra una terminal en el directorio del proyecto y ejecute el siguiente comando. Esto leerá el `Dockerfile` y construirá una imagen llamada net-analyzer.

docker build -t net-analyzer .

Paso 3: Ejecución de la Aplicación

3

Ejecute el contenedor con el siguiente comando. Esto iniciará la captura de paquetes y mostrará las estadísticas al finalizar.

docker run --rm --net=host -v $(pwd)/data:/app/data net-analyzer
